# /ops-playbook — 운영 매뉴얼 & 개발 운영 가이드

일일/주간/월간 루틴, CS 매뉴얼, 자동화 포인트, 개발 운영(DevOps) 가이드를 설계합니다.

## 실행 지시

1. 업종, 고객 수, 가용 시간(주당), 기술 제품 여부 파악
2. **시간 배분 프레임워크**: 주 10~15시간 기준 — CS/콘텐츠/핵심업무/지표/월간리뷰
3. **업종별 일일/주간 루틴** 체크리스트
4. **개발 운영** (기술 제품인 경우):
   - 배포 사이클: 핫픽스(즉시)/패치(주1~2)/마이너(월1)/메이저(분기1)
   - 1인 Git 워크플로우: main ← feature/fix/hotfix 브랜치
   - 버그/이슈 관리: GitHub Issues + 라벨링 + 우선순위(P0~P3)
   - 모니터링: Sentry + UptimeRobot + 알림(Slack/카톡)
   - Claude Code 운영 패턴: 버그 분석, 성능 개선, 코드 리뷰, 리팩토링, 문서 작성
5. **CS 매뉴얼**: 채널 우선순위, 응답 시간 기준, FAQ 자동화
6. **자동화 포인트**: 반복 업무별 도구 + 절감 시간 테이블
7. 번아웃 방지: 주 15시간 초과 시 경고
## 산출물 끝에 반드시 포함

### 3관점 검증
- 🔴 비판적: 번아웃 위험 구간, 자동화 부족 영역, 운영 비용 과소 추정
- 🟢 긍정적: 효율적인 시간 배분, 자동화로 절감 가능한 시간
- ⚪ 중립적: 루틴의 현실성 (본업 병행 시), 도구 선택의 적절성
