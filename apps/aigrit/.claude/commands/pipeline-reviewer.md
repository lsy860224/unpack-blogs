# /pipeline-reviewer — 파이프라인 QA & 최종 검증

전체/부분 파이프라인 산출물을 교차 검증하고 Go/No-Go 판단 리포트를 생성합니다.
"악마의 대변인" 역할 — 숨어있는 문제를 찾아냅니다.

## 실행 지시

1. **산출물 수집**:
   - 동일 대화면 → 직접 참조
   - 다른 대화면 → 핵심 수치 수집 폼 제시 (TAM/SOM, 가격, 고객수, 비용, BEP, 경쟁사, 리스크, CAC)
   - 파일 업로드 있으면 활용
2. **파이프라인 완성도 체크**: 의사결정 시점별 필수/권장 스킬 실행 여부 (누락 시 🔴/🟡)
3. **교차 검증 7축**:
   - A. 숫자 일관성: SOM↔고객가정, 가격↔ARPU, 초기투자↔셋업비용, CAC↔CPA 등
   - B. 가정 충돌: 리스크↔실행계획, 타겟↔가격, 규제↔세팅, 브랜드↔콘텐츠 톤
   - C. 논리 모순: BEP 전 스케일업, 낙관>SOM, 무료 플랜 비용 누락, 전환율/이탈률/K-factor/예산/KPI 스킬간 불일치
   - D. 데이터 최신성: 6개월+ → ⚠️ 재확인
   - E. 실행 가능성: 기술/시간/자본/역량 갭
   - F. 경쟁 현실성: 3개월 내 복제 가능 차별화 경고
   - G. Red Flag: "나만 생각" 신드롬, 희망적 사고, 기능 과잉, 시장 과대, 비용 과소, 고객 부재 검증, 마케팅 부재
4. **이슈 분류**: 🔴 Critical / 🟡 Warning / 🟢 Info — 각각 문제→근거→영향→수정 액션→담당 스킬→의도된 차이 가능성
5. **스코어카드** (/60): 시장/경쟁/수익/실행/리스크/마케팅 6영역 × /10 + Rubric 기반 채점 + 업종별 N/A 환산
6. **Go/Pivot/No-Go**: 50~60=Go, 35~49=Conditional Go, 20~34=Pivot, 0~19=No-Go
7. 🔴 이슈는 "이 판단이 맞습니까?" 사용자 확인 필수

"문제 없습니다" 결론은 의심 — 최소 3개 🟡 Warning은 존재해야 현실적.
## 산출물 끝에 반드시 포함

### 3관점 검증 (리뷰 자체에 대한 메타 검증)
- 🔴 비판적: 이 리뷰에서 놓쳤을 수 있는 이슈, 스코어링의 주관성, 사용자 컨텍스트 부족
- 🟢 긍정적: 교차 검증으로 발견된 핵심 이슈의 가치
- ⚪ 중립적: 리뷰 자체의 신뢰도 (입력 데이터 품질에 의존)

### 다음 단계
- 🔴 Critical 이슈 → 해당 스킬 재실행
- ✅ Go → 다음 Phase 진행
- 🔄 Pivot → /idea-generator로 돌아가 방향 전환
- 💡 **판단 기준**: Go/No-Go는 참고용이며 최종 결정은 사용자의 몫입니다.
